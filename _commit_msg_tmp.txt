- 跟 BattleItem.txt 对齐：游戏的 "优品=紫(q=4)、极品=金(q=5)、珍品=红(q=6)"
  vs 早期代码里错位的 "精品/珍品" → 全仓重命名 (synth_readings.py /
  observation.py / streamlit_app.py / demo_scenarios.py 以及 test fixtures)
  并补全 q=6 红物三件道具 (珍品扫描 / 珍品估价 / 珍品均格)。
- compute_tool_roi 新增 player_warehouse_noise_std 参数（默认 σ=10 cells）。
  之前没带"总仓储空间"的 fallback 直接用固定 159，等于假设玩家完美眼力，
  导致工具 ROI = 0 显得无用。现在按高斯噪声模拟玩家眼估误差：
    σ=0  → ROI = 0.000  （完美眼力，工具确实没价值）
    σ=10 → ROI = 0.446  （现实玩家，55K silver 换回 24.5K 价值）
    σ=15 → ROI = 0.924  （新手，几乎回本）
  Streamlit ROI tab 暴露 σ 滑块 + 解释 info 框。
- avg_cells 输入加 st.info 块，明确说明 "2.9" vs "2.90" 在引擎里是
  不同观测（尾零 → 截断 → 件数大概率不整除 10）。
