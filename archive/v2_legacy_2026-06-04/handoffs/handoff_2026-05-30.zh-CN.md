### 道具与英雄机制

已确认 action / skill 语义：

- `100129` 随机抽检（2）：返回 item_id、品质、价值、shape、cells、local。
- `100136` 宝光四鉴：返回 runtime + 品质；可通过后续 shape reveal 合并。
- `100105` 良品扫描：蓝色总格。
- `100104` 普品扫描：白绿总格。
- `100124` 优品估价：紫色总价值。
- `100112` 优品均格：紫色均格，fixed32 float。
- `100100` 全库透视：返回全库轮廓，可结算前锁定总件/总格。
- `100134` 明镜之眼：返回全库品质；和伊森已知品质轮廓 runtime 集合匹配后，可结算前锁定全库品质 + 形状。
- 艾莎 `1001034/1001033/1001032/1001031`：分别映射白/绿/蓝/紫累计轮廓。
- 伊森 `1002081..1002085`：逐轮已知品质轮廓，R5 可完整轮廓。

已保留扩展口：

- `quality_sample_histogram`
- `quality_sample_mode`

用于后续索菲/加布里、宝光这类“抽样品质”建模，目前不参与 MC 硬过滤。

### 坐标/布局

关键文件：

已确认坐标公式：

```text
local_index = (行 - 1) * 10 + (列 - 1)
```

也就是 10 列、0 基左上角。带 `shape_code` 的 local 才作为坐标锚点；宝光/明镜这类 quality-only local 可能不是形状左上角，不能覆盖 shape reveal 的坐标。

`local_index=None` 基本确认是 protobuf 默认值 0，即左上角。

关键设计决策：

- 最深行只作为布局深度证据展示，不作为总格数硬下界。
- 因为布局可能有空洞，`max_row * 10` 可能大于真实总格。
- 公开/鉴影/抽检露出的 item-level 轮廓只转为“品质下界”，不误当成整桶总量。

## 重要工程决策

1. 不把最深行当总格硬下界。
2. 不把单个公开物品当整桶总量。
3. 道具建议是局内使用排序，不是购买推荐。
4. 明镜之眼是机制验证/高成本特殊道具，不作为常态推荐。
5. Fatbeans 解析只做只读。
6. Streamlit 是实验台，长期 UI 通过 runtime snapshot 解耦。
7. 原始游戏表和抓包 JSON 不提交。
8. 概率模型继续基于游戏表白盒抽样，不做 ML 拟合掉率。
