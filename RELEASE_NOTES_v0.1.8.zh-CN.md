# BidKing Hero Ref v0.1.8

发布日期：2026-06-13
源 commit：`a706bd5`（干净工作树）
相对上一个发布包 **v0.1.6-hotfix2.1**（commit `f9fca54`）的全部关键改动。

## 英雄支持
- 英雄名单扩展到完整 20 名：现在能识别并给出估价。其中 **艾哈迈德、艾莎为深度专项优化**，其余英雄为通用 / 粗接入（generic ref）。
- 接入玛丽亚（Maria）技能解析，并排查「金品为零」相关情况。

## 估价与准确度
- **白绿件数修复（重点）**：以前会把「未知别墅」「宝光四鉴」等公开 / 技能的随机品质展示当成确定的白绿件数，导致低品被少算、价格虚高、甚至第三轮因约束冲突突然不报价。现在这类随机揭示只作为「下限（floor）」，有精确来源时才锁定确定值。
- 艾莎格 / 件区间收束：高档格 → 总格目标对齐、残差档按 avg_cells 估格、R3+ 总格上界扩展（C2 band）。相对无优化有提升、零门禁退化。
- 新增「报价不劣化门禁」（good_regression），避免后续优化把本已正确的报价改差。
- 修复 0052 第三轮白绿 bridge 估价。
- 艾莎红品权重 shadow 诊断 + R1–R5 防守倍数提示（仅提示，不改报价）。

## 小地图
- 修复拉文等「裸品质揭示」造成的位置偏移（贴边渲染恢复正确）。
- 不再每隔一秒闪烁（渲染签名缓存）。
- 支持宝物（treasure）footprint 渲染；钉住的小地图在手动拖动后可脱离。

## 界面与稳定性
- 恢复静默启动（不再弹出命令行窗口）。
- 关闭不再报错崩溃（`wm` / TclError 守护）；自动退出前先保存界面偏好。
- 退出时卸载随包的 WinDivert 驱动 → 解压目录可正常删除。
- 重新打开界面时底部不再被遮挡（渲染后重新贴合布局）。
- 手填 ↔ 实时切换能正常收起到 mini。
- 艾莎「下一步」提示改为按已知品质动态生成（如「等待蓝品和紫品信息」「等待紫品或总格、总件信息」），不再机械推荐道具。
- 一批 UI 细节打磨。

## 后台推理与诊断
- 空闲且局面稳定时，自动把快速（fast）估算升级为完整（full）推理，结果更可靠。
- 导出诊断新增「每轮滚动快照」，反馈问题时更易复现定位。

## 性能
- exact-total avg_cells 快速路径与若干隐藏 fast path，降低部分轮次耗时。

## 其他
- 署名更新（作者 加菲_Barista / 协作 lemyes）。

## 已知遗留（计划 v0.2.0）
- 没有「总件揭示」技能的英雄（如艾莎），在偏大局面的早期轮次估价偏保守（count_prior 中心标定，属架构层面，影响面大、需充分验证）。
