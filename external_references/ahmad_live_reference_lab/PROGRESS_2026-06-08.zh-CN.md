## 2026-06-08 实战反馈后的调整

### 去除 v3 主口径混入

问题：

- Ahmad UI 之前在 `ref_v0` 缺总件/总格时会从 v3 后验拿 `total_item_count_range` / `total_cells_range`；
- 面板主显示在 `ref_v0` 不完整时会显示 `v3_fallback`；
- 红件/红格/红值部分曾混用 ref 与 v3，导致类似“2-3 件 / 3 格”的不一致读数。

处理：

- `src/ahmad_ref_engine.py` 不再从 v3 后验读取总件或总格；
- `tools/ahmad_live_panel_server.py` 主卡片只显示 `ref_v0`、manual、settlement review 或 standby；
- ref 不就绪时主三档显示 `-`，动作显示 `等待外援输入`；
- 新增 `red_value_range`，红品件数、格数、价值都由 `ref_v0` 自己输出；
- 详情区从“v3 对照”改为“参考”，显示外援状态、ref 决策区间、总值、红值、结算和备注。

后续可回归主线的发现：

- UI 主路径不能静默 fallback 到另一套模型，否则用户无法判断当前价格来自哪里；
- 对照字段应留在诊断层，不能和主推荐混排；
- 红件、红格、红值必须来自同一条推理口径，否则会出现数值互相矛盾。

### Minimap 形状/品质显示

问题：

- 之前同一 item 会先画 footprint 矩形，再叠一个同色圆点；
- 用户看到的是“格子和圆圈放一起”，无法区分真实轮廓和品质-only 线索。

处理：

- 有 `cells > 0` 且来源不是 quality-only 的 item，只画 footprint；
- `quality_only` / `quality_reveal` / `public_quality` / `quality_marker` 或无 cells 的线索才画圆点；
- 最新 2527 settled minimap 有 33 个 packet footprint，canvas smoke 中不再生成 oval；
- synthetic quality-only smoke 会生成 oval，保留无轮廓品质线索能力。

后续可回归主线的发现：

- “真实形状”和“品质线索”应使用不同视觉语义；
- 有 shape/footprint 时，品质应通过填色/边框表达，不应再叠 marker；
- 无轮廓宝光/公开品质只应是软线索，不生成 hard footprint。

### 待机与旧局显示

问题：

- Ahmad HTTP summary 的普通 stale 和 settled stale 阈值实际是 600 秒；
- 因此结算后旧局可在 UI 停留 10 分钟，和用户预期的 60 秒不符。

处理：

- `STALE_SNAPSHOT_SECONDS = 60.0`；
- `SETTLED_STALE_SECONDS = 60.0`；
- 超过 60 秒后返回 standby payload，主三档、红品区间、结算真值、小地图都清空；
- standby flags 显示 `待机` 与 `旧快照`。

后续可回归主线的发现：

- live overlay 应区分“结算复盘窗口”和“下一局待机窗口”；
- 旧结算可以短时显示，但不能长时间占住主界面；
- monitor restart/session-ahead/stale snapshot 应统一走 standby payload。

- 红品卡摘要从 `紫 ... · 金 ...` 改为 `紫件 ... · 金件 ...`，明确这是推断件数范围，不是输入均格回显；
