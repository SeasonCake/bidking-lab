## 用户意图

用户希望先做一版更小、更好看的 Ahmed 实时推理 UI，并主要参考外援计算器路线。当前主线 v3 另有窗口在推进，因此本实验线必须保持隔离，新增内容先全部放在 `external_references/ahmad_live_reference_lab`。

## 当前设计结论

- 默认不使用 OCR；OCR 只作为高级层 fallback。
- 实时输入优先读取现有 live/Fatbeans packet 输出。
- 核心推理先采用外援式组合枚举和均格可达性，再吸收 v3 的 evidence/guard 标签。
- UI 必须一线显示红品数量或红品数量区间。
- UI 目标是 Tkinter 桌面 overlay，风格贴近当前 `scripts/run_live_overlay.py`；HTML 只保留调试 API。
- 新图/活动图要按 map family fallback 并标记 prior drift，不能完全 unknown。
- 任何外援参考输出都不能直接写入 formal bid。
- 作者/来源按用户确认记录为 B站猫饭团子uu：`https://space.bilibili.com/1981353429`。
- 本支线 UI 修改与计算优化署名为 B站加菲_barista：`https://space.bilibili.com/88048665`。
