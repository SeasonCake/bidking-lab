# bidking-lab · 操作说明（用户手册）

> **读者**：正在使用 Streamlit 推断台的玩家。  
> **不是**开发路线图 — 工程进度见 [`PROGRESS.md`](../PROGRESS.md)；踩坑见 [`TROUBLESHOOTING.md`](../TROUBLESHOOTING.md)。

## 这个工具做什么？

把游戏里**信息面板**上的文字（紫品总格、金品均价、地图名等）变成**可计算的约束**，再给出：

- 各品质 bucket 的候选 `(总格, 件数)`
- **出价推荐**（MC 分布 P25–P90）
- 道具 ROI（实验 tab）

**不会**自动替你出价，也**不**读取或修改游戏内存。

## 一屏上手（推荐顺序）

```mermaid
flowchart LR
  A[① 侧栏选英雄与地图] --> B[② 填仓库总格数]
  B --> C{③ 读数从哪来?}
  C -->|最快| D[抓取当前屏幕 OCR]
  C -->|备用| E[粘贴截图 / 上传图片]
  C -->|补漏| F[读数 tab 手填]
  D --> G[④ 打开出价推荐 tab]
  E --> G
  F --> G
  G --> H[⑤ 看分布图与建议价]
```

| 步骤 | 在哪做 | 要点 |
|------|--------|------|
| ① | 侧栏「会话」 | 先选**地图类型**再选**具体地图**；换图会按规则清空部分读数 |
| ② | 侧栏「仓库」 | OCR **通常不会**填这一项，请对照游戏内仓库容量**手填** |
| ③ | 侧栏「抓屏」或读数 tab | 游戏窗口在前台；信息区在屏幕**左侧中部** |
| ④ | 顶栏 **出价推荐** | 需已选地图 + 仓库格数 + 至少一条有效读数 |
| ⑤ | 同页 | 默认 MC **3000** 次；可勾选后台推断 |

## 抓屏 OCR 流程

```mermaid
sequenceDiagram
  participant U as 你
  participant UI as bidking-lab
  participant OCR as RapidOCR

  U->>UI: 点「抓取当前屏幕」
  Note over U: 切到游戏窗口（可选倒计时）
  UI->>UI: 截取主屏左侧信息区
  UI->>OCR: 识别文字
  OCR-->>UI: 面板全文
  UI->>UI: 解析地图名与各品质读数
  UI-->>U: 自动填入读数 tab
  U->>UI: 补填仓库格数 → 看出价
```

应用首屏现在可以直接使用：手填、切换 tab 不等待 OCR。只有在**首次点击**抓屏、剪贴板或上传图片 OCR 时才按需加载模型，因此第一次识别可能较慢，同一会话内后续识别会更快。

## 四个主 Tab 分工

```mermaid
flowchart TB
  subgraph tabs [顶栏四个 Tab]
    R[读数输入]
    H[出价推荐]
    J[联合筛选]
    I[道具 ROI]
  end
  R -->|填 cells / 总价 / 均格| H
  R --> H
  H -->|MC 分布 + 建议| U[实战出价参考]
  J -->|top-5 联合候选| X[仓库组成解释]
  I -->|Ethan 默认 kit| T[道具性价比]
```

| Tab | 用途 |
|-----|------|
| **读数输入** | 紫/金/红/蓝/白各品质的格数、件数、总价、均格、巨物档 |
| **出价推荐** | 条件价值直方图、P25/P50/P75/P90、分析估算 |
| **联合筛选** | 仓库组成 top 候选、联合结果与独立候选差异解释 |
| **道具 ROI** | _leave-one-out_ 道具价值（与当前地图/英雄相关） |

## 读数字段怎么理解？（简表）

| 游戏里常见说法 | 填哪里 | 进 MC 吗？ |
|----------------|--------|------------|
| 紫品总格 / 占位 | 紫品 **cells** | ✅ |
| 紫品件数 | **count** | ✅ |
| 紫品总价 / 总价值 | **value_sum** | ✅ |
| 紫品均格 2.90 | **均格**（文本框） | ❌ 主要帮**枚举候选** |
| 紫品均价 | **均价** | ❌ 同上 |
| 巨物「1 个」/ 具体船名 | **巨物 band** / ★ 下拉 | MC 看**件数**；★ 看**格数**（枚举） |

> 设计原则：**MC 用硬约束**；**均格/均价**用来收窄「一共有几种分法」——见 README 影响矩阵。

## 常见问题（30 秒）

1. **OCR 识别了地图，读数没填上？** — 换一张更清晰的截图，或手填；展开侧栏「上次导入」看 OCR 原文。
2. **紫品均格有数但框是空的？** — 已修复；若仍遇到请重启 Streamlit。
3. **切 tab 后预览「无合法候选」但框里还有数？** — 下方预览只看枚举；**MC 仍用**总价/格数/件数等硬字段。见 [`TROUBLESHOOTING.md`](../TROUBLESHOOTING.md) #46–47。
4. **打开应用还需要等待 OCR 吗？** — 不需要；手填和 tab 浏览立即可用。第一次实际 OCR 会按需加载模型，因此该次识别略慢。
