## 关于"格子形状"在哪里

我们一开始希望在 `Item.txt` 找到每个物品的网格轮廓（2D 0/1 矩阵），但：

- col[14] 只有 3 种值，**不可能**是 1207 件物品各自不同的形状。
- 没有任何一列分布看起来像"以物品为单位的 2D 矩阵"。

更可能的真实位置：
1. **`model/items/modle_*.data.manifest`**（Unity AssetBundle）—— 体积小，可能里面有 prefab 的尺寸。
2. **`Tables` 里另外一张我们还没看到的表**——但 `data/raw/tables/` 已经齐了 11 张，可能就是没有。
3. **直接在 `Assembly-CSharp.dll.bytes`**（游戏逻辑 DLL）——也就是说形状是硬编码或者通过反序列化 prefab 得到的。

→ **结论**：格子形状这块**先放到 Layer 2 的"待定"清单**。Layer 1 完全不依赖它就能交付（item_id / 名字 / 价值 / 品质 / 类别 / 掉落池 / 地图组合）。Layer 2 真要做 Q3（柜子装得下吗）时，我们再回来看 prefab。
