## 最近一次实机事实

旧进程完整打一局后：

```text
Capture: source=windivert age=1.3s flows=1 sniffed=241 raw=1 accepted=0 session=-
WARN: live capture saw payload but no auction frames were accepted
```

当时 `BidKing.exe` 同时可见：

```text
198.18.0.1:* -> 8.133.195.27:10000
127.0.0.1:* -> 127.0.0.1:*
127.0.0.1:* -> 127.0.0.1:7897
```

解释：

- `sniffed=241`：WinDivert 权限和基础抓取有效。
- `raw=1`：主要问题在 process-flow 归因层，而不是模型或 UI。
- VPN / UU / TUN / system proxy 下真实流量可能经过 loopback，本轮已默认纳入 loopback。
- 这只是代码修复，尚未完成重启后的实机验证。

## 计数解释

| 状态 | 说明 | 下一步 |
|---|---|---|
| `sniffed=0 / raw=0` | 没看到系统 TCP payload | 查 UAC、WinDivert、驱动和网络 |
| `sniffed>0 / raw≈0` | 有系统包，但没归因到目标流 | 查 loopback、代理进程、tuple 关联 |
| `raw>0 / accepted=0` | 抓到目标流，但 frame gate 没识别对局帧 | 增加受限 ignored-frame 诊断 |
| `accepted>0 / session=-` | 已放行帧但未建立当前局 | 查开局 `0x0021`、状态 `0x0025` 解析 |
| `accepted>0 / session!= -` | capture 基础链路已通 | 复核 snapshot、UI、道具、结算和日志 |
