## 4. 关键原则：不要过拟合几百局样本

当前真实样本规模是几百局、约一千多 ready windows，不是万级样本。后续不能把目标变成“把当前 archive 指标调到最好看”。

必须坚持：

- 先按结构化证据、概率机制和业务口径设计，再用样本验证。
- archive 指标只作为 falsification / regression evidence，不是唯一目标函数。
- 分层评估：地图族、英雄、轮次、信息密度、活动/普通、prior-stress、guarded/unguarded、formal/raw truth 分开。
- 样本少的 profile 不能升级成 formal，只能作为 shadow/watch。
- 任何参数如果只改善一个稀疏 cohort、同时伤害 holdout 或增加 extreme-over，应停止或降级为 audit-only。
- 实战参考价值优先：允许适度激进以减少严重 below，但不能让 P50/停止价被未验证长尾或弱先验带偏。
- 活动/版本漂移必须 fail-visible：缺表、活动、远端表不确定时，要显示 alias/fallback/source，而不是静默复用旧表。

## 5. 下一阶段长期项目项

这些是长期、按顺序推进的项目项，不是每个 turn 的微任务。

### A. 统一 v3 practical 评估合同

目标：让所有 archive/live/readiness 都能明确比较当前 UI 实战口径。

内容：

- 给 live/archive brief 增加显式 `formal_mode` 参数或 v3-practical 专用 replay。
- 分开统计 v2 formal、v3 practical unguarded、v3 practical guarded。
- guard 触发率、guard 后 MAE/below/P90 coverage/extreme-over、guard 是否误伤高 truth 局必须结构化输出。
- 所有评估必须标注 truth 口径：formal decision truth vs raw settlement truth。

成功标准：

- 不再出现“brief 看起来是 v3，实际是 v2 formal”的混淆。
- readiness 能看见 live guard 对 promotion metrics 的影响。

### B. prior/activity/table drift 鲁棒性收口

目标：让 v3 在活动、缺表、版本漂移时表现可解释、可回退。

内容：

- 继续区分普通 archive 与 activity cohort。
- 审查本地 raw tables、v303 `Activity/BidMap` 与仍缺失的 activity Drop/远端 overlay 表机制。
- 对 252x/452x activity alias 保持显式 fallback，不污染默认校准。
- prior-stressed rows 要按 map/profile 归因，决定是表漂移、活动、样本极端、还是证据不足。

成功标准：

- readiness 的 `prior_robustness` / `prior_stress_capacity_table_drift` 不再是模糊 blocked，而是明确哪些 cohort exclude、watch 或可建模。

### C. CSE/SCP/settlement bridge promotion hardening

目标：把另一个审计窗口遗留的 settlement over-cap / source semantics 后续项收口。

内容：

- CSE 与 SCP 继续保持 shadow/inactive。
- 跑 guarded settlement bridge 的 seed/session/map-family stability。
- 对 holdout hurt 的 bridge 直接降级或关闭，不继续靠参数挽救。
- 保留 capacity/source expansion 作为 risk/watch，不进入 formal value sampler。

成功标准：

- `settlement_count_guarded_bridge_stability` 有明确 pass/fail。
- fail 时 readiness 能说明关闭原因，避免下个窗口重复审计。

### D. evidence-driven count/cell/value sampler 重构

目标：做一个 shadow-only sampler 原型，让证据决定 q6/非 q6 的 count/cell/value 分布如何移动，而不是固定乘 prior 或简单 floor。

设计方向：

- 输入：ConstraintSet、FeasibleSummary、公开信息、道具结果、layout/item/shape/quality anchors、public random avg、known floors/exacts、capacity/source flags。
- 输出：count、cells、value 的联合 posterior，而不是分散的 q6_count floor、q6_cells floor、q6_value floor。
- 似然项必须可解释：value exact/floor、cells exact/floor、random sample avg、quality-only、shape footprint、warehouse total、settlement capacity/source。
- p50 与 p90 分开建模：P90 可以携带长尾风险，P50 不能被未验证尾部拉偏。
- 先做 shadow-only，严禁直接替换正式出价。

成功标准：

- 在 archive/session/seed holdout 上，证明其比当前 CCV/residual/formal-value sampler 更稳定。
- 尤其要看 directional hurt、below、normal absolute error、pinball、extreme-over。
- 若无法稳定优于 practical guard baseline，则保留为审计成果，不升 formal。

### E. live/UI 实战参考产品化

目标：让用户在实战中看得懂“不确定性来自哪里”，而不是只看到一个价格。

内容：

- UI 显示 guard 原因、weak-support、activity alias、public numeric facts、known floors。
- 不做视觉重做，只补必要信息。
- live restart/归档/样本整理流程保持简单。
- v2 fallback 必须保留。

成功标准：

- 用户能区分：正式停止价、P90 风险参考、弱证据 guard、活动 fallback、v2 对照。

### F. promotion 与 v2 archive 决策

目标：在有足够证据后再决定 v3 promotion，而不是因为 v2 差就仓促宣布完成。

promotion 前置条件：

- v3 practical/guarded 口径在 archive/readiness/live replay 一致。
- formal baseline metrics 不再 blocked。
- 关键 shadow lanes either pass holdout or explicitly disabled。
- sampler 若仍 shadow-only，不能成为 promotion 的隐式依赖。
- v2 fallback 可用，v2 archive 只在 promotion 后做。
